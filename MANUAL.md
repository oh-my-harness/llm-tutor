# Tutor Agent 使用手册

> 适用版本：0.2.1 及当前 `main` 分支
>
> 产品阶段：单用户、本地优先的桌面应用原型
>
> 文档状态：已按 `main` 分支截至 2026-07-14 的实现更新

Tutor Agent 是一个面向学习、研究和知识整理的本地 AI 工作区。它把多轮对话、复杂问题求解、联网研究、测验、知识库、Markdown 笔记本和学习记忆放在同一个桌面应用中。

本手册主要介绍当前版本已经可以使用的功能。尚未实现但需要帮助用户理解产品方向的内容会明确标记为“规划中”，不应当作当前稳定能力依赖。

## 1. 首次配置

首次启动后，先打开左侧边栏底部的“设置”。至少需要配置一个 LLM，其他服务按使用场景选配。

### 1.1 配置 LLM

进入“设置 > LLM”：

1. 点击“添加配置”。
2. 填写便于识别的配置名称。
3. 选择接口模式：`OpenAI-compatible` 或 `Anthropic Messages`。
4. 填写模型 ID、API Key、Base URL 和 Chat path。
5. 填写模型上下文窗口 token 数。
6. 点击“测试配置”。

可以保存多个 LLM 配置。聊天输入框和记忆工作区都可以在运行前选择具体模型。

OpenAI-compatible 模式可用于 OpenAI、DeepSeek 以及提供兼容 `/chat/completions` 接口的服务。不同服务的 Base URL、Chat path 和模型名称并不相同，应以服务商提供的信息为准。

### 1.2 配置嵌入模型

rag知识库入库和检索需要嵌入模型，**如果不需要rag知识库可跳过**。

进入“设置 > 嵌入模型”：

1. 添加 OpenAI-compatible embedding 配置。
2. 填写 Base URL、Embeddings path、API Key 和模型 ID。
3. 填写向量维度。
4. 按服务要求决定是否发送 `dimensions` 参数。
5. 使用测试按钮确认接口和维度可用。

知识库创建时会绑定一个嵌入配置。创建后不能直接把已有索引切换到另一个嵌入模型；需要使用新配置重新创建索引。

### 1.3 配置联网搜索

进入“设置 > 搜索”添加搜索配置。当前支持：

- DuckDuckGo
- Bing
- Brave
- Tavily
- Serper
- SerpAPI
- Exa

DuckDuckGo 可作为无需 API Key 的基础选项，但稳定性和结果质量可能波动。Research 模式建议配置稳定的搜索服务。

### 1.4 选择主题和语言

进入“设置 > 外观”：

- `冷灰浅色`：默认浅色主题。
- `石墨深色`：中性石墨深色主题。
- 界面语言：中文或英文。

主题切换立即生效并持久化，不会重置当前对话。

### 1.5 配置 Notebook 目录

进入“设置 > 笔记本”：

- 不绑定外部目录时，使用应用数据目录内的本地 Notebook。
- 桌面端可点击“选择文件夹”，绑定已有 Markdown Vault。
- 可导入 Markdown 文件或 zip。
- 可导出普通备份 zip 或 Obsidian 风格 Vault zip。

绑定外部 Vault 后，生成内容保存时会使用系统原生保存对话框，并限制目标位于该 Vault 内。

## 2. 界面结构

### 2.1 左侧主导航

当前主导航包括：

- 聊天
- 辅导机器人
- 知识库
- 笔记本
- 空间
- 记忆
- 设置

### 2.2 辅导机器人

“辅导机器人”是能够长期陪伴学习过程的持久导师身份，而不是另一个普通聊天模式。

当前可以创建、编辑和归档导师，并从聊天初始界面选择导师。未选择时默认使用不带持久身份的临时助手。导师身份会随会话保存，不能在已有会话中直接替换。

每位导师使用 Markdown Soul 定义稳定的身份、教学风格、专业方向、原则和边界。当前学习目标、下一步计划和未完成事项不写入 Soul，后续由独立 Tutor Memory 持续维护。

当前版本已经让 Soul 影响导师在 Chat、Research、Quiz 和 Deep Solve 中的教学方式。导师可指定默认模型、允许的能力和知识库，并控制 Notebook、Space 与学习者记忆访问；这些限制由服务端在会话创建、配置变更和运行时强制执行。独立 Tutor Memory、导师连续性面板和导师交接仍在开发中。

预期能力包括：

- 新建会话时可以选择一位持久导师；机器人代表“由谁陪伴学习”，Chat、Research、Quiz 和 Deep Solve 代表它当前使用的能力。
- 每个机器人拥有独立 Soul、默认配置、资料权限和会话集合；承诺、开放事项、课程计划、反思和教学策略将在 Tutor Memory 阶段加入。
- 获授权的机器人共享全局学习者记忆，但不会彼此共享私有计划和承诺；普通一次性问题仍可使用不带私有记忆的“临时助手”。
- 与用户共同明确学习目标、当前范围和阶段计划，并持续跟踪进展。
- 读取用户可见、可编辑的学习记忆，结合知识掌握情况、常见错误、偏好和教学策略调整讲解方式。
- 使用用户选择的知识库、Notebook 和空间内容作为学习材料，并保留可追溯的来源。
- 根据任务调用聊天、解题、研究、测验和整理能力，组织连续的学习流程，而不是要求用户手动拆分每一步。
- 结合测验结果和历史学习活动识别薄弱概念，安排针对性练习、复习和后续检查。
- 跨会话延续学习上下文，在用户离开后仍保留目标、进度和待办，但重要计划与记忆变更保持用户可见、可确认。

辅导机器人与普通聊天的主要区别在于“持续教学”：普通聊天面向当前问题，辅导机器人面向一段时间内的学习目标、策略、行动和反馈闭环。机器人不是一种新的对话模式，而是串联会话、能力、资料和记忆的持久身份。

### 2.3 最近会话

左侧“最近”区域显示历史会话：

- 点击会话项整行可打开会话。
- 当前打开的会话会显示选中标记。
- 正在后台运行的会话会显示运行标记。
- 鼠标移入后可重命名或删除。
- 右键可置顶/取消置顶、重命名或删除。
- 单纯选择会话不会改变排序；发送新消息或收到新活动后，未置顶会话才会前移。

### 2.4 右侧运行轨迹

进入已有会话后，聊天区右侧可显示 Agent 运行轨迹，包括思考阶段、工具调用、工作流进度、完成和错误状态。轨迹栏可以收起。

## 3. 聊天与会话

### 3.1 新建和继续会话

点击左侧“聊天”进入新对话。发送第一条消息后会创建持久会话。历史会话可从左侧“最近”重新打开。

会话保存的内容包括普通消息、模式、部分运行状态、引用信息，以及 Quiz 和 Research 等结构化附件。

### 3.2 输入框工具

输入框底部提供：

- 模式选择
- 文件附件
- Knowledge/Notebook 来源关联
- `@` 空间内容引用
- LLM 模型选择
- 发送/停止按钮

按 `Enter` 发送，按 `Shift + Enter` 换行。Agent 运行时，发送按钮会变成停止按钮。

### 3.3 文件附件

单个附件最大 16 MB。常见文本文件直接读取，PDF 和其他可解析文件交给后端解析。文本内容超过 20,000 字符时会截断并显示提示。

支持的常见文本扩展名包括：

`txt`、`md`、`csv`、`json`、`log`、`rs`、`ts`、`tsx`、`js`、`py`、`toml`、`yaml`、`xml`、`html`、`css`、`sql`。

### 3.4 关联 Knowledge 或 Notebook

来源选择器可把当前会话关联到：

- 一个知识库
- Notebook
- 不关联来源

关联知识库并不意味着每次都强制检索。Agent 会在需要时调用 `rag_search`，只有实际检索后才会产生对应引用。

### 3.5 `@` 引用空间内容

点击输入框中的 `@ Space`，可搜索并引用：

- Notebook 条目
- Quiz 会话
- Quiz 问题

引用会作为结构化目标随用户消息保存。Agent 使用前会读取对应内容，而不是只依赖标题。

### 3.6 消息操作

将鼠标移到消息上或使用键盘聚焦后，会出现操作栏：

- 复制
- 引用到输入框
- 编辑用户消息
- 重新生成支持该操作的回答
- 展开来源
- 将 Research 内容保存到 Notebook

普通 Agent 回答使用与聊天区域融合的内容面；用户消息显示为右侧灰色气泡。

## 4. 对话模式

### 4.1 聊天

适用于普通问答、解释、讨论和基于当前上下文的学习交流。聊天支持流式输出，也可以按需调用 RAG、网页搜索、网页抓取、代码执行和记忆读取工具。

### 4.2 解题（Deep Solve）

适用于需要分步骤分析、证据或计算验证的复杂问题。运行过程会显示计划、步骤和状态，完成后生成结构化答案。

可以针对某个解题步骤继续追问。代码执行是 Agent 可调用的工具，当前输入框中没有单独的“代码模式”入口。

### 4.3 测验（Quiz）

Quiz 模式保留普通聊天能力。Agent 可以先确认主题、范围、难度、题量和来源，再通过 `create_quiz` 启动独立测验工作流。

生成后的测验以可交互卡片显示：

- 逐题选择答案
- 提交答案并查看解释
- 前后切换问题
- 完成测验并记录结果
- 在空间的题库中重新打开

测验卡片与持久 Quiz 数据关联。切换会话再返回时，卡片应恢复并继续答题。

### 4.4 研究（Research）

Research 模式同样保留普通流式聊天。Agent 不会因为进入 Research 模式就强制开始调研，可以先通过对话确认：

- 研究主题和范围
- 时间范围
- 输出格式和深度
- 来源偏好
- 是否使用 Notebook 或知识库材料

确认后，Agent 通过 `create_research_report` 启动独立研究 workflow。工作流负责搜索、抓取、整理、综合和报告生成，主 Agent 负责对话与触发。

完成后的报告包含摘要、关键发现、分析、限制、后续问题和来源，并作为结构化报告附件显示。报告可保存到 Notebook，也可把网页来源导入知识库。

### 4.5 整理（Organize）

整理模式用于维护 Notebook。Agent 可以搜索笔记、读取条目并提出编辑、链接、标签或合并建议。建议需要用户确认后才会应用。

非整理模式可以读取明确引用或关联的 Notebook 内容，但不应主动发起 Notebook 维护提案。

## 5. 知识库与 RAG

### 5.1 创建知识库

创建前先配置并测试嵌入模型。进入“知识库”：

1. 点击新建知识库。
2. 填写名称。
3. 选择嵌入配置。
4. 创建知识库。

嵌入模型和维度会绑定到知识库索引。

### 5.2 上传和入库

知识库支持上传 PDF 和文本材料，也支持直接输入文本。入库过程包括解析、切分、嵌入和写入 LanceDB。

界面会显示上传和入库任务状态。大文件处理需要一定时间，请等待任务完成后再检索。

### 5.3 检索与引用

知识库页面可直接测试检索。聊天时，在输入框来源选择器中选中知识库，Agent 即可按需调用 RAG。

引用可包含知识库、文档、chunk 和匹配分数等元数据。点击可导航的引用时，应用会打开对应知识库区域。

### 5.4 删除

可以删除知识库中的文档或整个知识库。删除会影响后续检索，执行前应确认没有依赖这些材料的工作。

## 6. 笔记本

Notebook 使用 Markdown 作为主要持久格式，可用于保存研究报告、聊天答案、来源摘录、测验总结和解题结果。

### 6.1 文件树

- 新建笔记
- 新建文件夹
- 展开或收起文件夹
- 选择、编辑和删除笔记
- 重新进入页面时保持当前会话内的展开状态

文件夹初始默认收起，以减少长目录的视觉噪声。

### 6.2 阅读和编辑

选择条目后可阅读渲染后的 Markdown，也可进入编辑模式修改标题和正文。Markdown 支持常见标题、列表、表格、代码块、公式和链接。

Notebook 支持 `[[Wiki Link]]` 风格链接、反向链接和缺失链接创建入口。

### 6.3 保存生成内容

Research 报告等生成内容可从消息操作栏保存到 Notebook：

- 应用本地 Notebook 或浏览器开发模式使用应用内文件树选择目标文件夹。
- 桌面端绑定外部 Vault 后使用系统原生保存对话框。
- 默认文件名优先使用结构化标题，而不是正文第一句话。

### 6.4 外部 Vault

绑定外部 Vault 后，应用可刷新目录并监听外部 Markdown 变化。若怀疑文件监听遗漏，可手动点击刷新。

导入前会显示预览、冲突和跳过项。导出可选择备份 zip 或 Obsidian Vault zip。

## 7. 空间

空间聚合跨模块学习资产。当前包含：

### 7.1 题库

- 按来源筛选 Quiz
- 查看测验和问题
- 查看答案、解释和引用
- 删除测验
- 从引用跳转到对应来源

来源会区分 Knowledge Base、Notebook、Conversation 和 Space，而不是统一标记为 Knowledge。

### 7.2 学生画像

学生画像读取 L3 记忆中的画像、近期状态和教学策略。可以查看并编辑相关 Markdown 文件。

## 8. 记忆

记忆用于保存学习相关的长期上下文，不应被当作外部事实来源。

### 8.1 三层结构

- L1 工作区证据：聊天、测验、Notebook 和知识库原始事件。Research 模式下的需求确认与计划对话归入聊天事件，并保留 `capability = research`。
- L2 各模块摘要：`chat.md`、`quiz.md`、`notebook.md`、`knowledge.md`。已保存的 Research 报告由 `notebook.md` 归纳，不再使用独立 `research.md`。
- L3 跨模块记忆：近期状态、学生画像、学习范围、学习偏好和教学策略。

L1 当前不提供独立事件浏览器，记忆页面主要用于管理 L2 和 L3 Markdown。

### 8.2 查看和编辑

进入 L2 或 L3 后：

- 左侧选择记忆文件。
- 中间使用“阅读”模式查看渲染后的 Markdown。
- 切换到“编辑”模式后编辑 Markdown。
- Agent 产生候选变更后，中间区域自动进入“审核”模式。
- 点击保存写入文件。

### 8.3 LLM 工作区

右侧 LLM 工作区提供三种维护模式：

- 更新：从标准化来源中提取新的长期记忆。
- 检查：检查内容一致性和质量。
- 去重：提出重复内容的行级编辑。

先选择模式和已配置的 LLM，再点击运行。选择模式或模型本身不会立即启动 workflow。

新任务会在启动时采用当前全局界面语言。中文界面下，新生成的 L2/L3 记忆正文、摘要、检查发现和修改原因使用中文；英文界面下使用英文。切换语言不会翻译已有记忆，也不会改变已经开始运行的任务。代码、API、模型和论文名称等术语会按需要保留原文。

运行时采用明确的分层证据链：

- L2 任务读取对应模块的 L1 原始事件，并用本次实际读取的事件引用支撑变更。
- 普通 L3 任务先列出、搜索和读取稳定的 L2 条目，变更引用采用 `memory:L2/<file>#<entry-id>` 形式；只有核验条目来源时才沿引用追溯 L1。
- `recent.md` 是例外：它可以读取受限范围内的近期 L1 事件，以保留跨模块时间顺序，同时结合当前 L2 上下文。

Research workflow 的搜索、抓取、过程 trace 和未保存报告不会进入记忆。Research 对话通过 Chat L1/L2 进入长期归纳；只有用户把报告保存到 Notebook 后，报告内容才成为 `notebook.md` 可读取的证据。右侧工作区只显示排队、发现来源、读取证据、分析、生成变更和校验等进度，不显示整篇文档草稿。

当前 L3 已按目标文件限制可读取的 L2 来源，并校验普通变更是否引用了本次实际读过的 L2 条目。L2 新鲜度提示、应用变更时的来源版本校验，以及 `teaching_strategy.md` 严格等待已接受的 Profile、Scope、Preferences 后再生成，仍在完善。
运行期间可点击“取消运行”终止本次工作流。
运行期间可以切换到其他功能页面；返回“记忆”后，工作区会自动重新接入最新的运行中任务，或者打开已经生成的待审核变更。
左侧记忆文件栏会持续标记各模块的任务状态：旋转标记表示仍在运行，琥珀色圆点表示变更正在等待审核。多个模块有任务时会分别标记，切换页面不会清除这些状态。

Agent 返回结构化变更后，中间区域进入“审核”模式：

- 插入、替换和删除分别显示为确定性的差异块。
- 每项变更显示原因和本次运行实际读取过的来源。
- 可以逐项接受或拒绝，也可以全部接受或全部拒绝。
- 点击应用前不会修改记忆文件。
- 如果审核期间文件已被其他编辑修改，应用会停止并要求重新运行，避免覆盖新内容。
- 已接受的变更会一次性写入。应用前请确认差异内容；当前界面不提供“撤销上次保存”按钮。

### 8.4 Agent 如何使用记忆

- `read_memory`：在个性化教学、复习计划、测验难度和长期上下文需要时，读取当前应用数据目录中的 L3 记忆。Chat、Research、Quiz 对话和 Deep Solve 使用与记忆工作区相同的 Memory 目录。读取默认作为内部动作，不会在回答正文中播报“正在查看记忆”；Agent 会自然使用可靠的既往信息，对不确定记忆进行确认。工具调用仍可在运行轨迹中查看。
- `write_memory`：只保存用户明确要求记住或明确同意保存的长期偏好。
- 研究结果和外部事实应保存在 Research 报告、Notebook 或知识库，不应写成学习者记忆事实。

## 9. 设置

### 9.1 外观

切换主题和界面语言。主题选择立即保存。

### 9.2 LLM

管理多个对话模型配置、选择默认配置并测试连接。修改模型服务类设置会影响之后的新运行。

### 9.3 嵌入模型

管理知识库使用的 embedding 接口和维度。

### 9.4 搜索

管理 Agent `web_search` 工具使用的搜索服务、结果数量、抓取超时和最大正文字符数。

### 9.5 笔记本

绑定 Vault、导入、导出和查看 Notebook 状态。

### 9.6 能力

- Session budget：单次会话运行预算，单位为 USD。
- Require approval：工具执行前弹出批准对话框。
- 本地数据目录：桌面端可查看并打开实际数据目录。

## 10. 后台任务与会话恢复

Quiz、Research 和部分 Agent 任务可能运行较长时间。运行期间可以切换到其他会话：

- 原会话继续在后台运行。
- 左侧会话项显示运行标记。
- 返回原会话后，前端按稳定的 session/run 标识重新接入。
- 完成后的 Quiz 卡片或 Research 报告会附着到原会话。
- 快速切换会话时，旧会话事件不会写入当前新会话。

当前限制：应用进程重启后可以恢复持久消息和已完成附件，但正在执行的 workflow 尚不保证从中断点继续运行。此时会显示为中断或需要重新发起。

## 11. 本地数据与安全

### 11.1 数据目录

应用使用操作系统的应用数据目录保存本地内容。可在“设置 > 能力 > 本地数据目录”查看并打开准确位置。

主要数据包括：

```text
settings.json
sessions/
knowledge-bases.json
quizzes.json
notebook/
memory/
rag/
workflow-sessions/
```

### 11.2 API Key

当前版本会把 LLM、Embedding 和 Search 配置写入本地 `settings.json`。API Key 尚未接入系统钥匙串，因此：

- 不要共享整个应用数据目录。
- 不要把 `.llm-tutor/` 提交到版本控制。
- 公共或多人共用电脑上不要保存高权限 Key。
- 定期在服务商后台轮换 Key。

### 11.3 备份

至少备份 Notebook 和 `settings.json`。外部 Vault 可使用常规文件备份工具；应用本地 Notebook 可从设置中导出 zip。

## 12. 常见问题

### 12.1 模型无法回复

1. 打开“设置 > LLM”。
2. 确认模型 ID、API Key、Base URL 和 Chat path。
3. 点击“测试配置”。
4. 检查服务余额、网络和接口兼容性。
5. 在聊天输入框确认选择了正确模型。

### 12.2 Research 没有开始工作流

Research 模式允许普通对话，不会强制自动调研。清楚说明“开始详细调研”以及主题、范围和输出要求；Agent 完成必要确认后才会调用研究 workflow。

### 12.3 Research 只有过程文字，没有报告

查看右侧 trace 是否出现 `create_research_report`、搜索失败或模型错误。检查 Search 和 LLM 配置。完成的结构化报告应以报告附件渲染，而不是把所有工具过程拼成普通消息。

### 12.4 知识库检索不到内容

1. 确认入库任务已经完成。
2. 在知识库页面直接测试查询。
3. 确认聊天输入框关联了正确知识库。
4. 检查知识库绑定的 embedding 配置是否仍可用。

### 12.5 Notebook 外部文件没有更新

在 Notebook 或设置页面执行手动刷新。确认绑定目录仍存在且应用有读写权限。

### 12.6 任务运行中切换会话后看不到结果

查看左侧原会话是否仍有运行标记，并重新进入原会话。结果会写回发起任务的会话，而不是当前正在查看的会话。

## 13. 当前限制

- 当前是单用户本地产品，没有账号、权限和多人协作。
- 辅导机器人已支持持久身份、Soul、默认模型和服务端资料权限；私有 Tutor Memory、连续性面板和导师交接尚未实现。
- 自动更新和系统钥匙串尚未实现。
- 运行中的 workflow 在应用进程重启后不能保证续跑。
- RAG 切分和引用质量仍在持续改进。
- Memory 的 L2 新鲜度提示、应用时来源版本校验，以及教学策略的完整依赖顺序仍在完善。
- 原 Books 能力已经移除；Research 报告的持久化位置是 Notebook。
